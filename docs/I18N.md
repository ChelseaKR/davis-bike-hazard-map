# i18n status: IN-SCOPE — G10-static live · G9 blocked-on-catalog

`davis-bike-hazard-map` is a public civic map/list UI and is **in scope** for
internationalization per `INTERNATIONALIZATION-STANDARD.md` §1 (row
`davis-bike-hazard-map` — "Public civic map") and §11 ("pseudolocale + RTL gates
on the map UI; first gates to land: G9, G10").

This document records which of the standard's frontend depth gates are live,
which are blocked, and what is deferred. It follows the validated
`personal-site` reference pilot (ChelseaKR/personal-site PR #13).

## Current i18n state (as of this change)

The UI is **English-only today**: the React map/list surface renders hardcoded
English strings and there is **no message catalog** — no i18next / react-intl /
FormatJS, no `src/i18n/locales/` (or `lang/`) directory, no `useTranslation` /
`formatMessage` call sites. `src/lib/format.ts` uses raw `Intl.*`
(`DateTimeFormat`, `RelativeTimeFormat`) for dates/times, whose CLDR/tzdata come
from the pinned Node/V8 runtime, not an npm ICU dependency.

Consequence for the two frontend depth gates:

- **G10 (logical-CSS, static)** is **independent of the catalog** — it operates on
  the stylesheet, so it is implemented and **live** now.
- **G9 (pseudolocale overflow)** requires a catalog to pseudo-localize. With no
  catalog, there is nothing to transform into `en-XA`, so **G9 is
  blocked-on-catalog** (see below). This is a **declared decision**, not a silent
  skip.

## AUTO-GATES

Wired into the npm `verify` script (`npm run lint:css`, part of `npm run verify`
= the `make verify` merge gate) and the CI `verify` job (`.github/workflows/
ci.yml`) so **local == CI**, per §4.

| Gate | What it enforces | Mechanism | Status |
|---|---|---|---|
| **G10** Logical-CSS (static) | Inline-axis layout CSS uses **logical properties** (`inset-inline-*`, `margin-inline-*`, `padding-inline-*`, `border-inline-*`, logical border-radius corners, `text-align: start\|end`); physical `left/right`, `margin/padding/border-left\|right` are merge-blocking | `stylelint.config.js` (`csstools/use-logical`) via `npm run lint:css` | **live — passing** |
| **G9** Pseudolocale overflow | Key views render under an `en-XA` pseudolocale (~40% expansion) with no clipping / truncation / horizontal scroll | Playwright overflow spec + deterministic pseudo generator | **blocked-on-catalog** (needs Phase-1 extraction — see below) |

### G10 — logical-CSS (static live; ar/he RTL mirror deferred)

`stylelint.config.js` runs `csstools/use-logical` (plugin `stylelint-use-logical`)
scoped to the **inline (writing-direction) axis** — the axis that flips under RTL.
Block-axis (`top/bottom`, `margin/padding/border-top|bottom`) and box sizing
(`width/height`, `overflow-x/y`) are intentionally `except`ed: they do not mirror
under RTL, so converting them would be churn with no RTL benefit and is out of
G10's scope.

Only `src/**/*.css` (the app's own stylesheet, `src/styles.css`) is linted.
**Leaflet's bundled CSS lives in `node_modules` and is out of scope** — its
map-control glyphs/positioning are governed by Leaflet's own RTL handling, not
this gate, so no per-control disable is needed.

Conversions applied to `src/styles.css` in this change: **14 physical
inline-direction declarations** flagged by the gate were converted to logical
equivalents —

- `.skip-link` / `.skip-link:focus` off-screen positioning: `left` →
  `inset-inline-start` (×2);
- hazard/report/route severity accent bars: `border-left-width` /
  `border-left-color` → `border-inline-start-width` / `border-inline-start-color`
  (×10) — the severity accent correctly moves to the reading-start edge under RTL;
- `.route-hazard` accent border: `border-left` → `border-inline-start` (×1);
- `.route-steps` list indent: `padding-left` → `padding-inline-start` (×1).

Plus **1 paired directional declaration** converted for RTL consistency on an
element already being converted: `.route-hazard`'s `border-radius: 0 r r 0`
(rounds only the inline-end corners, opposite its accent border) → logical
corners (`border-start-start-radius` / `border-start-end-radius` /
`border-end-end-radius` / `border-end-start-radius`). Not flagged by the rule
(shorthand), but left physical it would round the wrong corners under RTL.

**LTR rendering is unchanged**: logical properties resolve to the identical
physical values in `direction: ltr` / `writing-mode: horizontal-tb`. **No
directional-icon or Leaflet-control exception was required** in this stylesheet
(unlike the personal-site pilot's one play-glyph exception).

> **Deferred (locked decision):** the **dynamic** half of G10 — the `ar`/`he`
> `dir=rtl` Playwright **mirror smoke** (INTERNATIONALIZATION-STANDARD §8) — is
> deferred until an RTL locale actually ships (which itself waits on the catalog,
> below). The static stylelint gate keeps new physical inline CSS from landing in
> the meantime, so adopting RTL later is a catalog + QA task, not a CSS-refactor
> task.

### G9 — pseudolocale overflow (blocked-on-catalog)

G9 asserts that the UI does not clip, truncate, or force horizontal scroll when
strings grow ~40% (the failure mode that surfaces the moment a longer locale —
Spanish, German — renders). It works by transforming the reference catalog into a
test-only `en-XA` pseudolocale and injecting it into the running i18n instance at
runtime.

**This repo has no catalog to transform, so G9 cannot be implemented yet.** It is
blocked on a **Phase-1 extraction** that must land first:

1. **Extract** the hardcoded UI strings behind an i18n call. Per
   INTERNATIONALIZATION-STANDARD §3, the TS/React entry point is either
   `react-intl` (`formatMessage`) or i18next (`useTranslation`); the personal-site
   reference uses i18next JSON catalogs (`src/i18n/locales/en.json`).
2. **Ship** `src/i18n/locales/en.json` (+ `es.json` for the EN/ES parity gate G6),
   with a single source of truth for supported languages (`src/i18n/config.ts`).
3. **Then** replicate the validated G9 infra from the pilot:
   - `scripts/i18n/gen-pseudo.mjs` — dependency-free, deterministic transform of
     `en.json` → `en-XA` (accent + ~40% expansion + `⟦ … ⟧` bracketing,
     protecting interpolation tokens); artifact written **outside**
     `src/i18n/locales/` and git-ignored so the catalog gates ignore it.
   - Expose the i18n instance on `window.__i18nTest` **only** behind a build-time
     flag (`VITE_I18N_TEST_HOOKS=1`), never in the production bundle.
   - A Playwright overflow spec that injects + activates the pseudo catalog and
     asserts `scrollWidth ≤ clientWidth` on the **UI chrome** (header, tabs,
     list, filters, report form, panels, controls) across key views × desktop +
     mobile. **Scope the assertion to the chrome, not the Leaflet map tiles** —
     the `.map-canvas` container has its own scroll and must be excluded.
   - Wire the spec into the normal e2e/unit structure (NOT the nightly
     `e2e-webkit` job).

Until then, G9 is recorded here as **blocked-on-catalog** and is not part of the
merge gate.

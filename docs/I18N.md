# i18n status: IN-SCOPE — catalogued (react-intl) · G1/G2/G3/G5/G6/G9/G10/G12 live · ES owed (R3)

`davis-bike-hazard-map` is a public civic map/list UI and is **in scope** for
internationalization per `INTERNATIONALIZATION-STANDARD.md` §1 (row
`davis-bike-hazard-map` — "Public civic map") and §11 ("pseudolocale + RTL gates
on the map UI; first gates to land: G9, G10").

A prior change (PR #39) landed the static half of **G10** (logical-CSS) and
recorded **G9** as blocked-on-catalog. **This change builds the Phase-1 message
catalog and unblocks the catalog-dependent gates.**

## Framework (INTERNATIONALIZATION-STANDARD §2/§3)

**react-intl / FormatJS** (`react-intl` ^7, `@formatjs/intl` ^3, `@formatjs/cli`
for extraction) — the standard's canonical stack for **new** TS/React work and
the MF2 migration path. `formatMessage` / `<FormattedMessage>` / `defineMessages`
with **stable, explicit ids**; no ICU MF1-only patterns (§9). An `<IntlProvider>`
(`src/i18n/IntlProviderShell.tsx`) mounts at the app root with browser-based
locale `negotiate()` (`src/i18n/config.ts`), an `en` default, and a
`useLocale()` lang-switcher hook for a future selector.

## Catalog

- **Source of truth for languages:** `src/i18n/config.ts` (`SUPPORTED_LANGUAGES`
  = `en`, `es`). Add a locale here + a JSON file to ship it.
- **`src/i18n/locales/en.json`** — reference catalog, **188 messages**, generated
  by `formatjs extract --format simple` (`npm run i18n:extract`). Flat
  `{ id: message }`, directly consumable by `IntlProvider`.
- **`src/i18n/locales/es.json`** — **structure-only**: every English id present
  with an **empty** value (`npm run i18n:extract` runs `scripts/i18n/sync-es-skeleton.mjs`).
  **No machine translation** — davis had no prior Spanish, and §5 R3 forbids
  unreviewed MT in civic prod. At runtime the loader drops empty values so react-intl
  falls back to the inline English `defaultMessage` (gettext-style). Users see
  correct English until Spanish is authored; nothing renders blank.
- **Enum labels** (hazard category / severity / lifecycle / 311 hand-off stage /
  device-queue state) live in `src/i18n/labels.ts` via `defineMessages`, so the
  framework-free `shared/types.ts` (also imported by the server) stays clean.

### Spanish is owed — REVIEW-GATE R3

### G9 — pseudolocale overflow (now live)

`scripts/i18n/gen-pseudo.mjs` transforms `en.json` → a git-ignored
`tests/i18n/en-XA.generated.json` (accents + ~40% expansion + `⟦ … ⟧`), protecting
ICU `{…}` argument/plural/select syntax **and** react-intl `<tag>` rich-text markup
so the pseudo message stays valid ICU. The catalog is injected into the running
provider via `window.__i18nTest`, exposed **only** when the build sets
`VITE_I18N_TEST_HOOKS=1` (the `playwright.i18n.config.ts` build does; the
production deploy build does **not** — verified: `__i18nTest` is dead-code-
eliminated from `dist/`). `tests/i18n/pseudo-overflow.spec.ts` asserts
`scrollWidth ≤ clientWidth` on the **UI chrome** (header, tabs, filters, list,
panels, forms, controls) across all seven views on **desktop + Pixel-7 mobile**.
The Leaflet map surface (`.map-canvas` / `.location-picker-map` / `.leaflet-container`)
is **excluded** — its tiles/controls pan and scroll by design. It runs in the CI
`e2e` job (chromium), **not** the nightly `e2e-webkit` job.

## Server-side surfaces (FIX-11 — localized via stable codes)

Server-visible surfaces are localized without shipping prose the client can't
translate:

- **API error envelopes.** Every error reply carries a stable machine `error`
  code (`server/app.ts`); `message` is only an English fallback for API
  consumers. The client translates off the code via `src/i18n/apiErrors.ts`
  (`apiError.<code>` catalog ids + a generic `apiError.unknown` fallback), never
  off `message`. A fine-grained `outside_davis` code (carried by a Zod
  refinement's `params.code`, see `shared/validation.ts`) lets the client
  distinguish an out-of-bounds location from a generic `validation_error`.
  `ApiRequestError.code` exposes the code; `.message` is now a diagnostic string.
- **New gate G2b (`i18n:shared-strings`).** `scripts/i18n/check-shared-strings.mjs`
  ratchets user-facing string literals in `shared/` (Zod `message:` / `.regex()`
  messages). The four existing validation messages are grandfathered (each backed
  by a stable code + a translated catalog id); any NEW one fails until it is
  mapped to a code. Wired into `npm run i18n:gates`.
- **Static policy pages.** `public/es/privacy.html` and
  `public/es/accessibility.html` ship (`lang="es"`), with reciprocal
  `<link rel="alternate" hreflang>` tags and a language-switch link on all four
  pages. Served by the existing `@fastify/static` mount of the client `dist/`
  (Vite copies `public/` verbatim).
- **Push payloads.** `AlertSubscription` carries an optional `locale` (validated
  against `ALERT_LOCALES`, default `en`); `buildAlertPayload(hazard, locale)` in
  `server/lib/pushNotify.ts` uses an en/es string table and the notify loop
  builds one payload per subscriber locale. The client sends its negotiated
  locale on subscribe (`src/lib/push.ts` → `subscribeAlert`).

> **⚠️ Spanish here is machine-drafted and PENDING NATIVE-SPEAKER REVIEW**
> (REVIEW-GATE **R3**). This covers the `apiError.*` es catalog values, the
> `public/es/*.html` pages (marked with an in-file comment), and the `es` push
> string table (marked with an in-file comment). None is final copy; a native
> reviewer must sign it off (`initial → translated → reviewed → final`) before
> launch. English remains the authoritative fallback until then.

## Deferred / out of scope for this change (honest ledger)

- **`src/lib/format.ts`** relative-time / distance / duration helpers
  (`timeAgo`, `formatDistance`, `formatDuration`) remain English-only, locale-free,
  and unit-tested. They produce user-facing phrases and are a **scoped follow-on**:
  migrating them to `Intl.RelativeTimeFormat` / `Intl.NumberFormat` (§2) is a
  separate change with its own acceptance tests, not silently skipped. The
  surrounding UI words that embed them ("Reported", "filed", "Updated", "Location:")
  **are** wrapped, with the helper output passed as an ICU placeholder.
- **G11** server negotiation (Phase 3, above).
- **G10** dynamic RTL mirror + **R2** (until an RTL locale ships).

Accessibility statement

We aim to meet WCAG 2.2 AA. Accessibility is a release gate: every build runs automated checks (axe) on components and full pages with zero violations allowed, and keyboard completion of the primary tasks is tested end to end.

Key commitments

  • The map is never the only way in — an equal, fully keyboard- and screen-reader-operable List view shows the same data.
  • Severity is conveyed by shape and text, never colour alone.
  • Visible focus, semantic landmarks, and a skip link throughout.
  • prefers-reduced-motion is honoured (animations reduced).
  • “No reports” is framed as under-reported, not “safe” — see the Coverage view.

Known limitations

A full screen-reader (VoiceOver/NVDA) sign-off is completed before public launch; the test script is published with the project. WebKit/Safari is covered by manual device testing.

Feedback

Found a barrier? Please contact the operator listed in the project README so we can fix it.
